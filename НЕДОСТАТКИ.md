# ECHO Messenger — Недостатки и что нужно сделать

> Составлено: 4 июля 2026  
> Статус: аудит текущего состояния приложения относительно тех. проекта v2.0

---

## 🔴 КРИТИЧЕСКИЕ — Сломано прямо сейчас

### 1. Реальное время (WebSocket) работает частично
- **Typing indicator** не отправляет WS-событие при наборе текста — бэкенд не рассылает его участникам чата
- **Статус «в сети»** в заголовке чата жёстко захардкоден строкой "в сети" — не отражает реальный онлайн-статус собеседника
- **Polling вместо push** — новые сообщения появляются через опрос каждые 3–5 сек, а не мгновенно через WS

### 2. Удаление и реакции не вызывают API
- Удаление сообщения показывает toast, но реально не удаляет запись в базе данных
- Реакции (эмодзи) показывают toast, но mutation на сервер не уходит — после перезагрузки реакций нет

### 3. Имена в групповых чатах
- Отправитель показывается как «User 123» вместо реального username
- Поле `senderUsername` не сохраняется в БД при отправке сообщения

### 4. Файлы и медиа
- Прикрепление файла через скрепку — показывает toast «в разработке», реально не отправляет
- Нет поддержки альбомов (несколько фото в одном сообщении)
- Нет превью (thumbnail) для видео и документов
- Нет индикатора прогресса загрузки

## 🟠 ВАЖНЫЕ — Функции заявлены, но не работают полноценно

### 5. Поиск
- Поиск работает только по username контактов и названиям чатов
- **Поиска по тексту сообщений нет вообще** — критически важная функция мессенджера
- Нет фильтрации результатов по типу (чаты / люди / медиа / ссылки)

### 6. Приватность — только UI-заглушки
- Настройки «Кто видит мой номер / фото / последний визит» существуют в UI, но не сохраняют реальные ограничения на сервере
- Настройка «Кто может мне писать» не влияет на API — любой может начать чат
- Пересылка сообщений не ограничивается флагом `isProtected` у чата

### 7. Секретные чаты — нет полноценного E2EE-ключевого обмена
- Секретный чат (тип 4) создаётся, но реального Signal-протокола между двумя клиентами нет
- Нет Diffie-Hellman key exchange через WS при открытии секретного чата
- `encryptedContent` хранится как есть — без реального шифрования на клиенте в demo-режиме

### 8. Голосовые сообщения
- Запись работает, отправка работает
- **Нет визуализации волновой формы** (waveform) — только иконка микрофона
- Нет переключения скорости воспроизведения (1× / 1.5× / 2×)
- Нет перемотки по тапу на прогресс-бар

### 9. Уведомления
- **Push-уведомлений нет вообще** — ни Service Worker, ни Web Push API не настроены
- При закрытой вкладке новые сообщения не приходят
- PWA-манифест есть, но `notificationclick` не обрабатывается

### 10. Звонки — частичная реализация
- Видеозвонки: UI экрана есть, WebRTC-инфраструктура есть, но сигнальный сервер через WS нестабилен при NAT
- Нет TURN-сервера — звонки работают только в одной сети (нет ICE candidates для relay)
- Входящий звонок иногда не появляется если вкладка неактивна

## 🟡 ОТСУТСТВУЮТ ПОЛНОСТЬЮ — По тех. проекту должны быть

### 11. Опросы (Polls)
- Схема БД создана (`polls` таблица существует)
- **UI нет, API-роутов нет, отправить опрос невозможно**
- Голосование, результаты, викторины — всё отсутствует

### 12. Папки чатов
- Схема БД создана (`chat_folders` таблица)
- **UI нет** — нет вкладок/папок в списке чатов как в Telegram
- Нет фильтров (Личные / Группы / Каналы / Непрочитанные / Без звука)

### 13. Геолокация
- Нет отправки геолокации в чате (ни live location, ни статическая точка)
- Схема для `locationLat/locationLng` в messages предусмотрена, но не задействована

### 14. Стикеры
- Страница стикер-паков есть, поиск есть
- **Реальных стикерпаков нет** — только пустые заглушки
- Нет установки пака, нет отправки стикера в чат
- Нет анимированных стикеров (Lottie)

### 15. Темы и Topics (форум-режим)
- Схема `topics` существует
- **Нет UI для форум-режима группы** — нельзя создать тему, нельзя переключаться между темами
- Нет флага `isForum` в интерфейсе настроек группы

### 16. Запланированные сообщения
- Поле `scheduledAt` в схеме сообщений есть
- **Нет UI для отложенной отправки** — нет кнопки «Отправить позже» в инпуте
- Нет экрана со списком запланированных сообщений

### 17. Напоминания о сообщениях
- Схема `message_reminders` создана
- **Нет возможности поставить напоминание** на сообщение (долгое нажатие → «Напомнить»)
- Нет бэкенд-планировщика (cron-job), который отправлял бы WS-уведомление в нужное время

### 18. Избранные сообщения (Saved Messages)
- Схема `saved_messages_stars` создана
- **Нет специального чата «Избранное»** — нет возможности сохранить сообщение в архив
- Нет страницы просмотра сохранённых сообщений

### 19. Анонимный инбокс (Ask-страница)
- Страница `/ask/:username` есть
- Не связана с основным чатом — ответить на анонимный вопрос из чата нельзя
- Нет уведомления получателю через WS при получении анонимного вопроса

### 20. Ссылки-приглашения в группы
- Поле `inviteLink` в схеме есть
- **Нет генерации invite-ссылки** в настройках группы
- `/invite/:link` роут есть, но бэкенд не генерирует эти ссылки

## 🔵 ТЕХНИЧЕСКИЙ ДОЛГ — Нужно сделать для продакшна

### 21. Безопасность
- **Пароли хранятся как SHA-hash** — нужен Argon2id (упомянуто в replit.md как «для demo»)
- Нет rate limiting на регистрацию — можно зарегистрировать тысячи аккаунтов
- Bearer-токен сессии не обновляется автоматически (нет refresh token механизма)
- CORS в Express не ограничен конкретным origin в production

### 22. База данных — неприменённые поля
Таблица `users` — поля есть в схеме, но **не возвращаются API и не используются в UI**:
- `isOnline` / `lastOnline` — статус в сети
- `isPremium` / `premiumUntil` — Premium-подписка
- `starsBalance` — баланс звёзд
- `bio` — описание профиля
- `displayName` — отображаемое имя

### 23. Производительность
- Сообщения загружаются все сразу — нет пагинации / infinite scroll
- Нет кеширования файлов (нет `Cache-Control` заголовков на `/api/files/:id/download`)
- Polling каждые 3 сек на всех открытых чатах — лишняя нагрузка на сервер

### 24. PWA
- `manifest.json` базовый — нет `share_target` (принять файл из другого приложения)
- Service Worker кешируется, но нет стратегии offline-first
- Нет экрана установки («Add to Home Screen» hint)

### 25. Светлая тема
- `next-themes` подключён, переключатель есть
- **Ряд элементов не адаптирован** под светлую тему (жёстко заданы тёмные hex-цвета в inline styles)
- Экран звонка (`call-screen.tsx`) полностью тёмный, игнорирует тему
