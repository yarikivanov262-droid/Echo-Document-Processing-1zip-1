---
name: Orval schemas conflict fix
description: Removing schemas from orval zod config prevents duplicate export conflicts; orval owns index.ts regeneration.
---

## Rule
Do NOT add `schemas: { path: "...", type: "typescript" }` to orval's `zod` output config. It generates TypeScript interfaces into a separate folder AND regenerates `lib/api-zod/src/index.ts` to export from both folders — causing TS2308 "already exported" errors when Zod schema names match interface names.

**Why:** Orval's `clean: true` + `schemas` option produces `generated/api.ts` (Zod schemas) AND `generated/types/` (TS interfaces) with identical export names (e.g. `EditMessageBody`). TypeScript cannot disambiguate them even with `export type *`.

**How to apply:** Keep only `target: "generated"` in the zod orval config. Zod schemas already embed full type info via `z.infer`. The `lib/api-zod/src/index.ts` file is regenerated by orval every codegen run — manual edits are overwritten; config is the only lever.
