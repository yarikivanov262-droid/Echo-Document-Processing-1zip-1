# ECHO — новая дизайн-система (ориентир: MAX / web.max.ru)

Текущий интерфейс ECHO выглядит устаревшим: голубой неоновый акцент, англоязычные подписи на экране входа, шрифт-моноспейс «хакерского» стиля. По присланным скриншотам MAX видно современное тёмное мессенджер-приложение с мягкими карточками, тёплым красно-оранжевым акцентом и человечным, а не «крипто-хакерским» тоном. Этот документ описывает новую дизайн-систему для ECHO в этом духе, сохранив тему приватности/безопасности как часть бренда, а не эстетику всего интерфейса.

---

## 1. Цветовая палитра

Сейчас: холодный голубой (`hsl(200 84% 55%)`) на чистом чёрном/белом фоне — выглядит как терминал, а не как приложение для людей.

Новая палитра (тёмная тема — основная, как в MAX):

| Токен | Значение | Применение |
|---|---|---|
| `--background` | `240 6% 7%` (почти чёрный с тёплым подтоном) | Фон приложения |
| `--card` | `240 5% 11%` | Карточки, поля ввода, всплывающие меню |
| `--card-elevated` | `240 5% 15%` | Модальные окна, шторки |
| `--primary` | `4 90% 58%` (тёплый красно-коралловый, как акцент MAX) | Основные кнопки, активные вкладки, исходящие сообщения (градиент) |
| `--primary-gradient` | линейный градиент `4 90% 58%` → `14 90% 62%` | Пузыри исходящих сообщений, FAB-кнопки |
| `--accent` | `280 60% 60%` (фиолетовый) | Второстепенные акценты — бейджи «Новое», значок «Сферум»-подобных разделов |
| `--destructive` | `0 72% 55%` | Удаление, блокировка, ошибки |
| `--muted-foreground` | `240 4% 62%` | Второстепенный текст (статус «был(а) недавно», подписи) |
| `--border` | `240 5% 18%` | Тонкие разделители |

Светлая тема — второстепенная, должна использовать ту же логику (тёплый акцент на светлом фоне), но приоритет — тёмная, т.к. это основной сценарий использования мессенджера.

**Важно:** акцент должен быть один тёплый цвет (красный/коралловый), а не голубой. Голубой ощущается как «системный», а не «фирменный».

## 2. Типографика

- Убрать `font-mono` с ключевых пользовательских экранов (сейчас на логине весь текст в моноспейсе — это создаёт ощущение «терминала/хакерской утилиты», а не мессенджера для обычных людей).
- Основной шрифт — тот же sans-serif, что уже используется в остальном приложении (Inter/system-ui), без изменений в остальных экранах.
- Моноспейс можно оставить только там, где это осмысленно: cid-хэши, seed-фраза при показе (уже используется в блоке из 12 слов на регистрации — это нормально), техническая информация в настройках безопасности.
- Заголовки экранов — полужирные, 17–20px, по центру шапки (как в MAX).
- Текст сообщений — 15–16px, межстрочный интервал 1.4.

---

## 3. Форма и скругления

- MAX использует крупные скругления: пузыри сообщений, карточки, поля ввода — радиус ~18-20px, а не «острые» углы.
- Обновить `--radius` с `0.75rem` на `1.1rem` (≈18px) для карточек и полей ввода; пузыри сообщений — `1.25rem` с «хвостиком»-подрезкой у последнего сообщения в группе (как в MAX/Telegram).
- Кнопки — полностью скруглённые (`rounded-full`) для основных CTA (как «DECRYPT VAULT» на логине сейчас — форма правильная, но стиль неправильный).

## 4. Компоненты

### Поля ввода (Input)
- Сейчас: тонкая рамка, прозрачный фон, иконка слева — выглядит как терминал.
- Новое: заливка `--card`, без видимой рамки в состоянии покоя, скругление 14px, placeholder приглушённым серым. Рамка/подсветка появляется только в фокусе — тёплым акцентным цветом.

### Кнопки (Button)
- Основная кнопка — заливка градиентом `--primary-gradient`, белый текст, без uppercase/tracking (сейчас везде `uppercase tracking-widest` — это тоже «хакерский» акцент, MAX использует обычный регистр).
- Вторичная кнопка — заливка `--card`, текст акцентного цвета.
- Деструктивная кнопка — красный текст на прозрачном/`--card` фоне (как «Удалить контакт», «Заблокировать» на скринах MAX).

### Строки настроек / списков (ToggleRow, ListItem)
- Уже структурно близко к MAX (карточка с разделителями, тумблер справа) — эту часть менять не нужно, только цвета тумблера: включено — тёплый акцент, а не системный зелёный/голубой (на скринах MAX тумблер красный в состоянии «вкл»).

### Аватары
- Круглые, с градиентной заливкой по умолчанию (если нет фото) — инициал по центру, как у контакта «R» на скрине MAX (розово-оранжевый градиент). Сейчас нужно проверить, что дефолтные аватары в ECHO используют похожую градиентную систему, а не однотонную заливку.

### Фон чата
- MAX использует едва заметный декоративный паттерн на фоне переписки (рыбки/иконки, полупрозрачные, ~5% opacity) поверх тёмного фона. Для ECHO можно сделать нейтральный минималистичный паттерн (например, точки или тонкие линии), не переусложняя.

### Пузыри сообщений
- Исходящие — заливка градиентом `--primary-gradient`.
- Входящие — заливка `--card`.
- Хвостик пузыря только у последнего сообщения в группе подряд идущих сообщений одного отправителя (как на скрине с сообщениями «Р»/«P» — сейчас видно, что все пузыри одинаковой формы без группировки).

## 5. Иконки и навигация

- Нижняя навигация (4 вкладки: Контакты / Звонки / Чаты / Настройки) — уже соответствует структуре MAX. Нужно только:
  - Активная вкладка подсвечивается тёплым акцентом (сейчас, вероятно, голубым).
  - Бейдж непрочитанных — красный кружок с числом (уже частично есть, проверить консистентность цвета с новой палитрой).
- Шапка чата: аватар + имя + статус слева, три иконки справа (видео, звонок, «ещё») — уже соответствует MAX, менять структуру не нужно, только визуальный стиль иконок (line-иконки, не filled).

## 6. Тон текста

- Экран входа сейчас звучит как «крипто-инструмент»: «Decrypt Vault», «Cryptographic Seed», «Initialize Generator», «Master Seed Generated». Для приложения, которое хочет быть понятным широкой аудитории (как MAX), тон должен быть проще и человечнее, оставаясь честным про приватность.
- Рекомендуемая замена тона (не дословный перевод, а стиль):
  - «Decrypt Vault» → «Войти»
  - «Generate new keys» → «Создать аккаунт»
  - «Cryptographic Seed» → «Кодовая фраза» (с пояснением ниже поля: «12 слов, которые заменяют пароль и email»)
  - «Master Seed Generated» → «Ваша кодовая фраза готова»
  - «Initialize Generator» → «Создать кодовую фразу»

Подробная раскладка по экранам — в файле `ДИЗАЙН_ЭКРАНЫ.md`.
