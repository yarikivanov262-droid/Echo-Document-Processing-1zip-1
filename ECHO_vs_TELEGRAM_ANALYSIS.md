# ECHO Messenger vs Telegram — Полный сравнительный анализ
> Версия документа: 5 июля 2026  
> Telegram версия: ~12.x (2026)  
> ECHO: текущая кодовая база

---

## Условные обозначения

| Символ | Значение |
|--------|----------|
| ✅ | Реализовано полностью |
| 🟡 | Частично / UI есть, логика не полная |
| ❌ | Отсутствует полностью |
| 🔵 | ECHO-эксклюзивная функция (нет в Telegram) |

---

## 1. НАВИГАЦИЯ И ОСНОВНЫЕ ВКЛАДКИ

| Функция | Telegram | ECHO | Примечание |
|---------|----------|------|------------|
| Вкладка «Чаты» (список) | ✅ | ✅ | Реализована как `/chats` |
| Вкладка «Контакты» | ✅ | ✅ | Страница `/contacts` |
| Вкладка «Звонки» | ✅ | ✅ | Страница `/calls` |
| Вкладка «Настройки» | ✅ | ✅ | Страница `/settings` |
| Вкладка «Избранное» (Saved Messages) | ✅ | ✅ | Страница `/favorites` (starred messages) |
| Вкладка «Секретные чаты» | ✅ | ✅ | Страница `/secret-chats` |
| Папки чатов (кастомные вкладки в sidebar) | ✅ | 🟡 | БД и страница есть, но UI папок как вкладок не реализован |
| Архив чатов | ✅ | 🟡 | API архивации есть, отдельной вкладки «Архив» нет |
| Stories (вверху чатов) | ✅ | ❌ | Отсутствует полностью |
| Быстрый переключатель аккаунтов (мультиаккаунт) | ✅ | 🟡 | Есть «Мои профили» / Personas, но переключение не multi-account в классическом смысле |

| Боковая панель (sidebar) — desktop | ✅ | ✅ | Sidebar реализован |
| Floating action button (новый чат) | ✅ | ✅ | Кнопка «+» для нового чата |

## ЭТАП 1 — Базовые UX-блокеры (критично, 1-2 недели)

- ❌→✅ Галочки доставки (✓ / ✓✓) в списке

### 1.3 Форматирование текста

- ❌→✅ «Иван добавлен в группу», «Мария вышла» — server-side events через WS

## ЭТАП 2 — Типы сообщений (2-3 недели)

### 2.1 Кружочки (Video Notes)
- Запись 60-секундного круглого видео через `getUserMedia({video: {aspectRatio: 1}})`
- Отправка как blob, отображение в круглом элементе с плеером

### 2.2 Геолокация
- Статичная — отправить координаты, показать Google Maps / OpenStreetMap тайл
- Live — передача через WS events `location_update`

### 2.3 Контакт
- Отправить карточку пользователя (имя + ECHO-номер/username)

### 2.4 Режим «view-once»
- Флаг `viewOnce: true` в message metadata
- После открытия — серверное удаление контента файла

### 2.5 Расписание отправки (Schedule)
- UI: «Отправить позже» → выбор даты/времени
- Backend: сохранение со статусом `scheduled`, cron/job для отправки

### 2.6 Quiz-режим в опросах
- Поле `correctOptionId` в схеме poll
- UI: выделение правильного ответа после голосования

```sql
CREATE TABLE stories (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  media_file_id UUID REFERENCES files(file_id),
  type VARCHAR(10) -- 'photo' | 'video'

  created_at TIMESTAMPTZ DEFAULT now()
);
CREATE TABLE story_views (
  story_id UUID, viewer_id UUID, viewed_at TIMESTAMPTZ
);
CREATE TABLE story_reactions (
  story_id UUID, user_id UUID, emoji VARCHAR(10), created_at TIMESTAMPTZ
```

- Поле `autoDeleteAfterSeconds` в чате (для secret chats)
- Клиентский таймер + серверное удаление

### 12.3 Per-chat фон (wallpaper)
- Поле `wallpaper` в `chat_members.settings` JSONB
- UI выбора фона в настройках чата

### 12.4 Мут с таймером
- «Заглушить на 1ч / 8ч / 1 день / навсегда»
- Поле `mutedUntil: timestamp` в `chat_members`

### 12.5 Выбор нескольких сообщений (multi-select)
- Long-press → режим выделения
- Bulk: удалить / переслать / скопировать

### 12.6 Отправка без звука
- Флаг `silent: true` в сообщении
- Получатель не получает звук уведомления
