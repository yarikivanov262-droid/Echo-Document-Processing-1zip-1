================================================================================
         ECHO MESSENGER — ПОЛНЫЙ ТЕХНИЧЕСКИЙ ПРОЕКТ v2.0
         «Говори. Никто не узнает.»
         Аналог Telegram. Без телефона. Без email. Только seed-фраза.
================================================================================

СТЕК:
  Node.js 24 + Express 5 + PostgreSQL + Drizzle ORM
  React 18 + Vite + Tailwind CSS + Framer Motion + Wouter + TanStack Query
  WebSocket (ws lib) + WebRTC (нативный браузер API)
  Монорепо pnpm workspaces

СТРУКТУРА МОНОРЕПО:
  artifacts/api-server/         — Express backend (порт 8080)
  artifacts/echo-messenger/     — React PWA frontend (порт 5000)
  lib/db/src/schema/            — Drizzle ORM таблицы
  lib/api-spec/openapi.yaml     — OpenAPI контракт (источник истины)
  lib/api-client-react/         — Сгенерированные TanStack Query хуки
  lib/api-zod/                  — Сгенерированные Zod-схемы

=== STARS & PREMIUM API (создать artifacts/api-server/src/routes/stars.ts) ===

GET /api/stars/balance
  SELECT stars_balance FROM users WHERE id = userId
  Ответ: { balance: number }

GET /api/stars/transactions
  SELECT * FROM stars_transactions WHERE from_user_id = userId OR to_user_id = userId ORDER BY created_at DESC
  Ответ: [{ id, type, amount, description, fromUserId, toUserId, createdAt }]

POST /api/stars/gift
  Body: { toUserId: number, amount: number }
  Проверить что users.starsBalance >= amount
  BEGIN TRANSACTION:
    UPDATE users SET stars_balance = stars_balance - amount WHERE id = userId
    UPDATE users SET stars_balance = stars_balance + amount WHERE id = toUserId
    INSERT INTO stars_transactions (from, to, amount, type: "gift")
  COMMIT
  Broadcast WS к toUserId: { type: "stars_received", amount, fromUsername }
  Ответ: { success: true, newBalance: number }

POST /api/stars/purchase
  Body: { package: "100" | "250" | "500" | "1000" | "2500" }
  Пакеты цен (демо-режим, без реальной оплаты):
    "100" → +100 stars, стоимость 0 (демо)
    Реальная интеграция: проверить payment_id от платёжного провайдера
  UPDATE users SET stars_balance = stars_balance + :amount WHERE id = userId
  INSERT INTO stars_transactions (to_user_id: userId, amount, type: "purchase")
  Ответ: { success: true, newBalance: number }

POST /api/stars/tip
  Body: { messageId: number, amount: number }
  Найти senderId сообщения
  Списать stars у текущего пользователя, начислить автору
  INSERT INTO reactions_paid (message_id, user_id, stars_amount)
  Broadcast WS к chatId: { type: "paid_reaction", messageId, amount, fromUsername }
  Ответ: { success: true }

POST /api/premium/subscribe
  Body: { tier: "monthly" | "annual" }
  Цены: monthly = 280 Stars, annual = 2800 Stars
  Проверить баланс, списать Stars
  UPDATE users SET is_premium = true, premium_until = :date WHERE id = userId
  INSERT INTO premium_subscriptions
  Ответ: { success: true, expiresAt: string }

  import { createContext, useContext, useState, useCallback } from "react";
  import { useWsEvent } from "@/hooks/use-ws";

  type OnlineMap = Record<number, { isOnline: boolean; lastOnline: string | null }>;
  const OnlineCtx = createContext<OnlineMap>({});
  const OnlineSetCtx = createContext<(userId: number, data: OnlineMap[number]) => void>(() => {});

  export function OnlineProvider({ children }: { children: React.ReactNode }) {
    const [map, setMap] = useState<OnlineMap>({});
    const update = useCallback((userId: number, data: OnlineMap[number]) => {
      setMap(prev => ({ ...prev, [userId]: data }));
    }, []);

    useWsEvent((event) => {
      if (event.type === "status") {
        update(event.userId, { isOnline: event.online, lastOnline: event.lastOnline ?? null });
      }
    });

    return (
      <OnlineCtx.Provider value={map}>
        <OnlineSetCtx.Provider value={update}>
          {children}
        </OnlineSetCtx.Provider>
      </OnlineCtx.Provider>
    );
  }

  export const useOnlineStatus = (userId?: number) => {
    const map = useContext(OnlineCtx);
    if (!userId) return null;
    return map[userId] ?? null;
  };

  export function useVoiceRecorder() {
    const [isRecording, setIsRecording] = useState(false);
    const [duration, setDuration] = useState(0);
    const [waveform, setWaveform] = useState<number[]>([]);
    const mediaRecorderRef = useRef<MediaRecorder | null>(null);
    const chunksRef = useRef<Blob[]>([]);
    const timerRef = useRef<number | null>(null);
    const analyserRef = useRef<AnalyserNode | null>(null);
    const animFrameRef = useRef<number | null>(null);

    const startRecording = async () => {
      const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
      
      // Waveform analyser
      const audioCtx = new AudioContext();
      const source = audioCtx.createMediaStreamSource(stream);
      const analyser = audioCtx.createAnalyser();
      analyser.fftSize = 64;
      source.connect(analyser);
      analyserRef.current = analyser;
      
      const recorder = new MediaRecorder(stream, { mimeType: "audio/webm;codecs=opus" });
      chunksRef.current = [];
      recorder.ondataavailable = e => { if (e.data.size > 0) chunksRef.current.push(e.data); };
      recorder.start(100);
      mediaRecorderRef.current = recorder;
      setIsRecording(true);
      setDuration(0);
      timerRef.current = window.setInterval(() => setDuration(d => d + 1), 1000);
      
      // Update waveform
      const updateWave = () => {
        const data = new Uint8Array(analyser.frequencyBinCount);
        analyser.getByteFrequencyData(data);
        setWaveform(Array.from(data.slice(0, 30)).map(v => v / 255));
        animFrameRef.current = requestAnimationFrame(updateWave);
      };
      animFrameRef.current = requestAnimationFrame(updateWave);
    };

    const stopRecording = (): Promise<Blob> => new Promise(resolve => {
      if (!mediaRecorderRef.current) return;
      mediaRecorderRef.current.onstop = () => {
        resolve(new Blob(chunksRef.current, { type: "audio/webm" }));
      };
      mediaRecorderRef.current.stop();
      mediaRecorderRef.current.stream.getTracks().forEach(t => t.stop());
      if (timerRef.current) clearInterval(timerRef.current);
      if (animFrameRef.current) cancelAnimationFrame(animFrameRef.current);
      setIsRecording(false);
      setDuration(0);
      setWaveform([]);
    });

    const cancelRecording = () => {
      if (!mediaRecorderRef.current) return;
      mediaRecorderRef.current.stop();
      mediaRecorderRef.current.stream.getTracks().forEach(t => t.stop());
      if (timerRef.current) clearInterval(timerRef.current);
      if (animFrameRef.current) cancelAnimationFrame(animFrameRef.current);
      setIsRecording(false);
      setDuration(0);
      setWaveform([]);
    };

    return { isRecording, duration, waveform, startRecording, stopRecording, cancelRecording };
  }

=== USE-WEBRTC HOOK ===

  const ICE_SERVERS = [
    { urls: "stun:stun.l.google.com:19302" },
    { urls: "stun:stun1.l.google.com:19302" }
  ];

  export function useWebRTC({ callId, targetUserId, isInitiator, callType }) {
    const pcRef = useRef<RTCPeerConnection | null>(null);
    const [remoteStream, setRemoteStream] = useState<MediaStream | null>(null);
    const [localStream, setLocalStream] = useState<MediaStream | null>(null);
    const [callState, setCallState] = useState<"connecting"|"ringing"|"active"|"ended">("connecting");
    const [isMuted, setIsMuted] = useState(false);
    const [isVideoEnabled, setIsVideoEnabled] = useState(callType === "video");
    
    const init = async () => {
      const constraints = { audio: true, video: callType === "video" ? { facingMode: "user" } : false };
      const stream = await navigator.mediaDevices.getUserMedia(constraints);
      setLocalStream(stream);
      
      const pc = new RTCPeerConnection({ iceServers: ICE_SERVERS });
      pcRef.current = pc;
      
      stream.getTracks().forEach(t => pc.addTrack(t, stream));
      
      pc.ontrack = e => { if (e.streams[0]) setRemoteStream(e.streams[0]); };
      pc.onicecandidate = e => {
        if (e.candidate) {
          echoWs.send(JSON.stringify({ type: "call_signal", targetUserId, signal: { type: "ice-candidate", candidate: e.candidate } }));
        }
      };
      pc.onconnectionstatechange = () => {
        if (pc.connectionState === "connected") setCallState("active");
        if (["disconnected","failed","closed"].includes(pc.connectionState)) setCallState("ended");
      };
      
      if (isInitiator) {
        const offer = await pc.createOffer();
        await pc.setLocalDescription(offer);
        echoWs.send(JSON.stringify({ type: "call_signal", targetUserId, signal: { type: "offer", sdp: offer.sdp } }));
      }
    };
    
    const handleSignal = async (signal: Record<string,unknown>) => {
      const pc = pcRef.current;
      if (!pc) return;
      if (signal.type === "offer") {
        await pc.setRemoteDescription({ type: "offer", sdp: signal.sdp as string });
        const answer = await pc.createAnswer();
        await pc.setLocalDescription(answer);
        echoWs.send(JSON.stringify({ type: "call_signal", targetUserId, signal: { type: "answer", sdp: answer.sdp } }));
        setCallState("active");
      } else if (signal.type === "answer") {
        await pc.setRemoteDescription({ type: "answer", sdp: signal.sdp as string });
      } else if (signal.type === "ice-candidate") {
        await pc.addIceCandidate(signal.candidate as RTCIceCandidateInit);
      }
    };
    
    const toggleMute = () => {
      localStream?.getAudioTracks().forEach(t => { t.enabled = !t.enabled; });
      setIsMuted(v => !v);
    };
    
    const toggleVideo = () => {
      localStream?.getVideoTracks().forEach(t => { t.enabled = !t.enabled; });
      setIsVideoEnabled(v => !v);
    };
    
    const hangup = () => {
      localStream?.getTracks().forEach(t => t.stop());
      pcRef.current?.close();
      setLocalStream(null);
      setRemoteStream(null);
      setCallState("ended");
    };
    
    return { init, handleSignal, hangup, toggleMute, toggleVideo,
             localStream, remoteStream, callState, isMuted, isVideoEnabled };
  }

sw.js:
  const CACHE_NAME = "echo-v1";
  const STATIC_ASSETS = ["/", "/index.html", "/manifest.json"];

  self.addEventListener("install", e => {
    e.waitUntil(caches.open(CACHE_NAME).then(c => c.addAll(STATIC_ASSETS)));
    self.skipWaiting();
  });

  self.addEventListener("activate", e => {
    e.waitUntil(caches.keys().then(keys => Promise.all(
      keys.filter(k => k !== CACHE_NAME).map(k => caches.delete(k))
    )));
    self.clients.claim();
  });

  self.addEventListener("fetch", e => {
    if (e.request.method !== "GET") return;
    if (e.request.url.includes("/api/")) return; // не кэшировать API
    e.respondWith(caches.match(e.request).then(r => r || fetch(e.request)));
  });

  self.addEventListener("push", e => {
    const data = e.data?.json() ?? {};
    e.waitUntil(self.registration.showNotification(data.title ?? "ECHO", {
      body: data.body ?? "Новое сообщение",
      icon: "/icons/icon-192.png",
      badge: "/icons/badge-72.png",
      vibrate: [200, 100, 200],
      data: { url: data.chatUrl ?? "/" },
      actions: [
        { action: "open", title: "Открыть" },
        { action: "dismiss", title: "Закрыть" }
      ]
    }));
  });

  self.addEventListener("notificationclick", e => {
    e.notification.close();
    if (e.action === "dismiss") return;
    e.waitUntil(clients.openWindow(e.notification.data?.url ?? "/"));
  });

export function startBackgroundJobs() {
  // Самоуничтожение сообщений (каждые 10 сек)
  setInterval(async () => {
    const expired = await db.select({ id: messagesTable.id, chatId: messagesTable.chatId })
      .from(messagesTable)
      .where(and(
        lte(messagesTable.selfDestructAt, new Date()),
        eq(messagesTable.isDeleted, false),
        isNotNull(messagesTable.selfDestructAt)
      ));
    
    for (const msg of expired) {
      await db.update(messagesTable).set({ isDeleted: true }).where(eq(messagesTable.id, msg.id));
      broadcastToChat([/* member ids */], { type: "delete_message", messageId: msg.id, chatId: msg.chatId });
    }
  }, 10_000);

  // Напоминания (каждые 30 сек)
  setInterval(async () => {
    const due = await db.select().from(messageRemindersTable)
      .where(and(lte(messageRemindersTable.remindAt, new Date()), eq(messageRemindersTable.isDone, false)));
    
    for (const reminder of due) {
      sendToUser(reminder.userId, { type: "reminder", messageId: reminder.messageId });
      await db.update(messageRemindersTable).set({ isDone: true }).where(eq(messageRemindersTable.id, reminder.id));
    }
  }, 30_000);

  // Истечение Premium подписок (каждый час)
  setInterval(async () => {
    const expired = await db.select({ userId: premiumSubscriptionsTable.userId })
      .from(premiumSubscriptionsTable)
      .where(and(lte(premiumSubscriptionsTable.expiresAt, new Date()), eq(premiumSubscriptionsTable.isActive, true)));
    
    for (const { userId } of expired) {
      await db.update(usersTable).set({ isPremium: false, premiumUntil: null }).where(eq(usersTable.id, userId));
      await db.update(premiumSubscriptionsTable).set({ isActive: false }).where(eq(premiumSubscriptionsTable.userId, userId));
    }
  }, 3_600_000);
}

=== ЧТО ТАКОЕ ОБХОД БЛОКИРОВОК ДЛЯ МЕССЕНДЖЕРА ===

Когда провайдер блокирует ECHO:
  - DNS-запросы к домену ECHO → перенаправляются или блокируются
  - Прямые HTTP/WS соединения к API → сбрасываются (RST пакет)
  - Порты 80/443 → фильтруются по IP

Что мы можем сделать без внешних проксей (внутри самого ECHO):

  1. DOMAIN FRONTING через Cloudflare
  2. Встроенный SOCKS5/HTTP прокси (пользователь вводит свой)
  3. MTProto-like OBFUSCATION обфускация WebSocket
  4. Несколько fallback-доменов (резервные домены)
  5. Tor через встроенный WASM (Tor.js)
  6. DNS over HTTPS (DoH) — обход блокировки DNS
  7. WebRTC TURN-based соединение как транспорт
  8. i2p/Snowflake bridge (продвинутый уровень)

=== РЕАЛИЗАЦИЯ — ШАГ 1: CONNECTION MANAGER ===

Файл: artifacts/echo-messenger/src/lib/connection-manager.ts

  type ConnectionMode =
    | "direct"           // обычное соединение
    | "alt-domain"       // резервный домен
    | "cf-worker"        // Cloudflare Workers прокси
    | "custom-proxy"     // пользовательский прокси
    | "tor"              // Tor WASM

  type ConnectionConfig = {
    mode: ConnectionMode;
    proxyUrl?: string;       // для custom-proxy: "socks5://host:port" или "http://host:port"
    cfWorkerUrl?: string;    // URL Cloudflare Worker
    altDomains?: string[];   // список резервных доменов
    timeout: number;         // таймаут проверки соединения в мс
  };

  const DEFAULT_CONFIG: ConnectionConfig = {
    mode: "direct",
    altDomains: [
      "https://echo1.example.com",
      "https://echo2.example.com",
      "https://echo-backup.onion.ws"  // onion-через-HTTPS мост
    ],
    timeout: 5000
  };

  class ConnectionManager {
    private config: ConnectionConfig;
    private activeMode: ConnectionMode = "direct";
    private activeBaseUrl: string = "";
    private listeners: Array<(mode: ConnectionMode) => void> = [];

    constructor() {
      // Загрузить сохранённый конфиг из localStorage
      const saved = localStorage.getItem("echo_connection_config");
      this.config = saved ? JSON.parse(saved) : DEFAULT_CONFIG;
      this.activeBaseUrl = window.location.origin;
    }

    async autoDetect(): Promise<ConnectionMode> {
      // Пробуем прямое соединение
      if (await this.testConnection("direct", window.location.origin)) {
        this.setMode("direct");
        return "direct";
      }

      // Пробуем резервные домены
      for (const domain of this.config.altDomains ?? []) {
        if (await this.testConnection("alt-domain", domain)) {
          this.activeBaseUrl = domain;
          this.setMode("alt-domain");
          return "alt-domain";
        }
      }

      // Пробуем CF Worker
      if (this.config.cfWorkerUrl && await this.testConnection("cf-worker", this.config.cfWorkerUrl)) {
        this.activeBaseUrl = this.config.cfWorkerUrl;
        this.setMode("cf-worker");
        return "cf-worker";
      }

      // Пробуем пользовательский прокси
      if (this.config.proxyUrl) {
        this.setMode("custom-proxy");
        return "custom-proxy";
      }

      // Fallback: Tor
      this.setMode("tor");
      return "tor";
    }

    private async testConnection(mode: ConnectionMode, baseUrl: string): Promise<boolean> {
      try {
        const controller = new AbortController();
        const timeout = setTimeout(() => controller.abort(), this.config.timeout);
        const res = await fetch(`${baseUrl}/api/health`, {
          signal: controller.signal,
          cache: "no-store"
        });
        clearTimeout(timeout);
        return res.ok;
      } catch {
        return false;
      }
    }

    private setMode(mode: ConnectionMode) {
      this.activeMode = mode;
      this.listeners.forEach(cb => cb(mode));
    }

    onModeChange(cb: (mode: ConnectionMode) => void) {
      this.listeners.push(cb);
      return () => { this.listeners = this.listeners.filter(l => l !== cb); };
    }

    getBaseUrl(): string { return this.activeBaseUrl; }
    getMode(): ConnectionMode { return this.activeMode; }
    getConfig(): ConnectionConfig { return this.config; }

    saveConfig(cfg: Partial<ConnectionConfig>) {
      this.config = { ...this.config, ...cfg };
      localStorage.setItem("echo_connection_config", JSON.stringify(this.config));
    }
  }

  export const connectionManager = new ConnectionManager();

=== РЕАЛИЗАЦИЯ — ШАГ 2: ОБФУСКАЦИЯ WEBSOCKET ===

Проблема: провайдер видит WS handshake и блокирует по заголовкам.
Решение: обфускация — XOR-шифрование первых байт + случайный padding.

Файл: artifacts/echo-messenger/src/lib/obfuscated-ws.ts (создать)

  const OBFUSCATION_KEY = 0x5A; // простой XOR ключ (можно сделать случайным)

  function xorBytes(data: Uint8Array, key: number): Uint8Array {
    return data.map(byte => byte ^ key);
  }

  export class ObfuscatedWebSocket extends EventTarget {
    private ws: WebSocket;
    private enabled: boolean;

    constructor(url: string, obfuscate = false) {
      super();
      this.enabled = obfuscate;
      this.ws = new WebSocket(url);
      this.ws.binaryType = "arraybuffer";

      this.ws.onopen = () => this.dispatchEvent(new Event("open"));
      this.ws.onclose = (e) => this.dispatchEvent(new CloseEvent("close", e));
      this.ws.onerror = (e) => this.dispatchEvent(new Event("error"));

      this.ws.onmessage = (e) => {
        let data: string;
        if (this.enabled && e.data instanceof ArrayBuffer) {
          const decoded = xorBytes(new Uint8Array(e.data), OBFUSCATION_KEY);
          data = new TextDecoder().decode(decoded);
        } else {
          data = e.data as string;
        }
        this.dispatchEvent(new MessageEvent("message", { data }));
      };
    }

    send(data: string) {
      if (this.enabled) {
        const encoded = xorBytes(new TextEncoder().encode(data), OBFUSCATION_KEY);
        this.ws.send(encoded.buffer);
      } else {
        this.ws.send(data);
      }
    }

    close() { this.ws.close(); }
    get readyState() { return this.ws.readyState; }
  }

Backend — обработка обфусцированных сообщений:
  В ws-hub.ts при получении бинарного сообщения:
    ws.on("message", (data, isBinary) => {

  const DOH_PROVIDERS = [
    "https://cloudflare-dns.com/dns-query",
    "https://dns.google/dns-query",
    "https://dns.adguard.com/dns-query"
  ];

  export async function resolveHostname(hostname: string): Promise<string | null> {
    for (const dohUrl of DOH_PROVIDERS) {
      try {
        const res = await fetch(`${dohUrl}?name=${hostname}&type=A`, {
          headers: { "Accept": "application/dns-json" }
        });
        const json = await res.json() as { Answer?: { data: string }[] };
        const ip = json.Answer?.[0]?.data;
        if (ip) return ip;
      } catch {
        continue;
      }
    }
    return null;
  }

    STAGE 32 — НАСТОЯЩИЙ E2EE: SIGNAL PROTOCOL (DOUBLE RATCHET + X3DH)
    «Сервер не может прочитать ни одного сообщения физически»
================================================================================

ПРОБЛЕМА ТЕКУЩЕЙ РЕАЛИЗАЦИИ:
  encryptedContent = просто текст сообщения (не зашифрован!)
  Сервер хранит и видит все сообщения в открытом виде.
  Это противоречит заявленному E2EE.

ПРАВИЛЬНАЯ РЕАЛИЗАЦИЯ (Signal Protocol):

=== КОНЦЕПЦИЯ ===

  Каждый пользователь имеет:
    Identity Key Pair (IK)     — долгосрочный ключ идентификации
    Signed Prekey Pair (SPK)   — среднесрочный ключ (меняется каждые 7 дней)
    One-Time Prekeys (OPK)     — одноразовые ключи (пул из 100 штук)
  
  Все публичные ключи хранятся на сервере (уже есть поля в users таблице).
  Приватные ключи НИКОГДА не покидают устройство.

=== X3DH (Extended Triple Diffie-Hellman) — установка сессии ===

  Алиса хочет написать Бобу (первое сообщение):
  
  1. Алиса получает от сервера: IK_bob, SPK_bob, OPK_bob (один одноразовый)
  2. Алиса генерирует временную пару EK (Ephemeral Key)
  3. Алиса вычисляет:
       DH1 = DH(IK_alice_private, SPK_bob_public)
       DH2 = DH(EK_alice_private, IK_bob_public)
       DH3 = DH(EK_alice_private, SPK_bob_public)
       DH4 = DH(EK_alice_private, OPK_bob_public)  ← если OPK доступен
       masterSecret = KDF(DH1 || DH2 || DH3 || DH4)
  4. masterSecret → AES-256-GCM ключ для шифрования первого сообщения
  5. В первом сообщении Алиса прикладывает: IK_alice_public, EK_alice_public, OPK_id

  Боб при получении:
  1. Берёт свои приватные ключи и публичные ключи Алисы из сообщения
  2. Вычисляет те же DH1-DH4 → тот же masterSecret
  3. Расшифровывает сообщение

=== DOUBLE RATCHET — каждое сообщение новый ключ ===

  После X3DH оба имеют RootKey (RK).
  
  Symmetric-key Ratchet (Chain Key → Message Keys):
    CK_s_next, MK = KDF(CK_s)  ← Алиса каждый раз при отправке
    CK_r_next, MK = KDF(CK_r)  ← Боб каждый раз при получении
  
  Diffie-Hellman Ratchet (при ответе):
    Боб генерирует новую пару DH_bob_new
    RK, CK_r = KDF(RK, DH(DH_bob_new_private, DH_alice_public))
    Прикладывает DH_bob_new_public к сообщению
  
  Результат: каждое сообщение зашифровано уникальным ключом.
  Компрометация одного ключа не раскрывает ни прошлые ни будущие сообщения.
  Forward Secrecy + Break-in Recovery — оба свойства выполнены.

  // Генерация ключевой пары (X25519 / ECDH)
  async function generateKeyPair(): Promise<CryptoKeyPair> {
    return crypto.subtle.generateKey({ name: "ECDH", namedCurve: "P-256" }, true, ["deriveKey", "deriveBits"]);
  }

  // ECDH обмен
  async function dh(privateKey: CryptoKey, publicKey: CryptoKey): Promise<ArrayBuffer> {
    return crypto.subtle.deriveBits({ name: "ECDH", public: publicKey }, privateKey, 256);
  }

  // KDF (HKDF)
  async function hkdf(input: ArrayBuffer, salt: ArrayBuffer, info: string, length = 32): Promise<ArrayBuffer> {
    const importedKey = await crypto.subtle.importKey("raw", input, "HKDF", false, ["deriveBits"]);
    return crypto.subtle.deriveBits({
      name: "HKDF",
      hash: "SHA-256",
      salt,
      info: new TextEncoder().encode(info)
    }, importedKey, length * 8);
  }

  // AES-256-GCM шифрование
  async function encrypt(key: ArrayBuffer, plaintext: string): Promise<{ ciphertext: string; iv: string }> {
    const aesKey = await crypto.subtle.importKey("raw", key, { name: "AES-GCM" }, false, ["encrypt"]);
    const iv = crypto.getRandomValues(new Uint8Array(12));
    const encrypted = await crypto.subtle.encrypt(
      { name: "AES-GCM", iv },
      aesKey,
      new TextEncoder().encode(plaintext)
    );
    return {
      ciphertext: btoa(String.fromCharCode(...new Uint8Array(encrypted))),
      iv: btoa(String.fromCharCode(...iv))
    };
  }

  // AES-256-GCM расшифровка
  async function decrypt(key: ArrayBuffer, ciphertext: string, iv: string): Promise<string> {
    const aesKey = await crypto.subtle.importKey("raw", key, { name: "AES-GCM" }, false, ["decrypt"]);
    const ivBytes = Uint8Array.from(atob(iv), c => c.charCodeAt(0));
    const ctBytes = Uint8Array.from(atob(ciphertext), c => c.charCodeAt(0));
    const decrypted = await crypto.subtle.decrypt({ name: "AES-GCM", iv: ivBytes }, aesKey, ctBytes);
    return new TextDecoder().decode(decrypted);
  }

  const SHAKE_COUNT_REQUIRED = 5;
  let shakeCount = 0;
  let lastShake = 0;
  
  window.addEventListener("devicemotion", e => {
    const acc = e.acceleration;
    if (!acc) return;
    const magnitude = Math.sqrt(acc.x!**2 + acc.y!**2 + acc.z!**2);

=== ОЧИСТКА EXIF МЕТАДАННЫХ ===

При загрузке фото/видео — удалять все EXIF данные.
EXIF содержит: GPS координаты, модель камеры, время съёмки, серийный номер.

Реализация (без библиотек, через canvas):
  Файл: artifacts/echo-messenger/src/lib/media/strip-exif.ts
  
  export async function stripExif(file: File): Promise<File> {
    return new Promise(resolve => {
      const img = new Image();
      const objectUrl = URL.createObjectURL(file);
      img.onload = () => {
        const canvas = document.createElement("canvas");
        canvas.width = img.naturalWidth;
        canvas.height = img.naturalHeight;
        const ctx = canvas.getContext("2d")!;
        ctx.drawImage(img, 0, 0);
        URL.revokeObjectURL(objectUrl);
        canvas.toBlob(blob => {
          resolve(new File([blob!], file.name, { type: "image/jpeg" }));
        }, "image/jpeg", 0.95);
      };
      img.src = objectUrl;
    });
  }

При загрузке любого изображения → stripExif() перед base64 конвертацией.

=== ЗАШИФРОВАННОЕ ЛОКАЛЬНОЕ ХРАНИЛИЩЕ ===

IndexedDB (сессии, ключи, кэш):
  Мастер-ключ = PBKDF2(seedPhrase, userId, 200000 iterations) → AES-256-GCM
  При каждой записи: encrypt(data, masterKey) → сохранить зашифрованное
  При каждом чтении: decrypt(data, masterKey) → вернуть

  Файл: artifacts/echo-messenger/src/lib/secure-storage.ts
  
  class SecureStorage {
    private key: CryptoKey | null = null;
    
    async init(seedPhrase: string, userId: number) {
      const encoder = new TextEncoder();
      const baseKey = await crypto.subtle.importKey("raw", encoder.encode(seedPhrase), "PBKDF2", false, ["deriveKey"]);
      this.key = await crypto.subtle.deriveKey({
        name: "PBKDF2",
        salt: encoder.encode(`echo-${userId}`),
        iterations: 200000,
        hash: "SHA-256"
      }, baseKey, { name: "AES-GCM", length: 256 }, false, ["encrypt", "decrypt"]);
    }
    
    async set(key: string, value: unknown): Promise<void> {
      if (!this.key) throw new Error("SecureStorage not initialized");
      const iv = crypto.getRandomValues(new Uint8Array(12));
      const data = new TextEncoder().encode(JSON.stringify(value));
      const encrypted = await crypto.subtle.encrypt({ name: "AES-GCM", iv }, this.key, data);
      // Сохранить iv + encrypted в IndexedDB
      await idbSet(key, { iv: Array.from(iv), data: Array.from(new Uint8Array(encrypted)) });
    }
    
    async get<T>(key: string): Promise<T | null> {
      const stored = await idbGet(key);
      if (!stored) return null;
      const decrypted = await crypto.subtle.decrypt(
        { name: "AES-GCM", iv: new Uint8Array(stored.iv) },
        this.key!,
        new Uint8Array(stored.data)
      );
      return JSON.parse(new TextDecoder().decode(decrypted)) as T;
    }
  }
  
  export const secureStorage = new SecureStorage();

Инициализировать в auth-context.tsx после логина:
  await secureStorage.init(seedPhrase, userId);

=== P2P-DM РЕЖИМ (WebRTC DataChannel) ===

Если оба пользователя онлайн в WebSocket → можно передавать сообщения напрямую
через WebRTC DataChannel, минуя сервер.

Как это работает:
  1. Алиса и Боб оба подключены к WS
  2. Алиса инициирует P2P соединение (RTCPeerConnection с dataChannel)
  3. Через WS сервер выполняется signaling (offer/answer/ICE — как для звонков)
  4. После установки соединения — сообщения идут напрямую P2P
  5. Сервер не видит содержимое (уже зашифровано E2EE)

Backend всё равно нужен:
  - Для доставки когда один из пользователей offline
  - Для хранения истории (зашифрованной)
  - Для discovery (найти другого пользователя)
  - Для signaling P2P

Реализация:
  Файл: artifacts/echo-messenger/src/lib/p2p-channel.ts
  
  export class P2PChannel {
    private pc: RTCPeerConnection | null = null;
    private dc: RTCDataChannel | null = null;
    
    async initiate(targetUserId: number, signal: (data: object) => void) {
      this.pc = new RTCPeerConnection({ iceServers: ICE_SERVERS });
      this.dc = this.pc.createDataChannel("messages", { ordered: true });
      
      this.dc.onopen = () => console.log("P2P channel open");
      this.dc.onmessage = (e) => this.onMessage(e.data);
      
      this.pc.onicecandidate = e => {
        if (e.candidate) signal({ type: "p2p-ice", targetUserId, candidate: e.candidate });
      };
      
      const offer = await this.pc.createOffer();
      await this.pc.setLocalDescription(offer);
      signal({ type: "p2p-offer", targetUserId, sdp: offer.sdp });
    }
    
    sendMessage(encryptedMsg: string) {
      if (this.dc?.readyState === "open") {
        this.dc.send(encryptedMsg);
        return true; // успешно отправлено P2P
      }
      return false; // нужно слать через сервер
    }
    
    private onMessage(data: string) {
      // Получено P2P сообщение — обработать как обычное входящее
    }
  }

В sendMutation: сначала пробовать P2P, если канал не открыт → обычный API.

    STAGE 41 — ECHO НОМЕРА (+999XXXXXXX)
    «Коллекционный анонимный идентификатор»
================================================================================

=== КОНЦЕПЦИЯ ===

Каждый пользователь может получить уникальный ECHO Номер вида +9991234567.
Номер виден всем пользователям, привязан к профилю и не связан с реальным
телефонным номером. Это коллекционный identifier, аналог красивого номера в мессенджере.

Формат: +999 + 7 цифр = 10 цифр после +
Примеры: +9991234567, +9990000001, +9999999999

=== ОСОБЕННОСТИ ===

- Номера начинаются с +999 (несуществующий код страны — полная анонимность)
- Пользователь выбирает любой свободный номер самостоятельно
- Можно сменить номер в любое время (предыдущий освобождается)
- Номер виден в профиле и в поиске рядом с @username
- Поиск пользователей работает по номеру: введи +9991234567 → найди человека
- Номер выдаётся по схеме "первый занял — его"
- Красивые номера (+9990000001, +9990001337) — коллекционная ценность

=== DATABASE ===

lib/db/src/schema/users.ts:
  echoNumber: varchar("echo_number", { length: 16 }).unique()
  — nullable, уникальный индекс

=== API ===

GET  /api/numbers/check/:number     → { available: boolean, number: string }
POST /api/numbers/claim             → UserProfile (с echoNumber)
GET  /api/users/by-number/:echoNumber → PublicUser

=== BACKEND ===

Файл: artifacts/api-server/src/routes/numbers.ts
Валидация: /^\+999\d{7}$/ — строго 10 символов после +

=== FRONTEND ===

my-profile.tsx:
  - Отображает ECHO номер рядом с @username
  - Кликабельная строка "ECHO Номер" → открывает EchoNumberPicker bottomsheet
  - EchoNumberPicker: ввод номера + кнопка случайного номера (Shuffle)
  - Дебаунс 500мс при проверке доступности
  - Цветовая индикация: зелёный = свободен, красный = занят

search.tsx:
  - Определяет тип запроса: если начинается с +999 → поиск по номеру
  - GET /api/users/by-number/:number
  - Показывает ECHO номер рядом с @username в результатах
  - Плейсхолдер: "Поиск по @username или +999..."

=== USER STORY ===

1. Алиса открывает "Мой профиль" → видит "Получить номер +999..."
2. Нажимает → открывается пикер с случайным числом (+9997823441)
3. Может ввести желаемое (например +9990001337) → проверка в реальном времени
4. Нажимает "Привязать номер" → номер сохраняется
5. Боб видит её номер в профиле, может скопировать
6. В поиске вводит +9990001337 → мгновенно находит Алису
