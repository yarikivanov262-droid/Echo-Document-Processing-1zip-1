# ECHO — что не работает и что нужно доделать

Документ создан на основе анализа кода фронтенда (`artifacts/echo-messenger`) и бэкенда (`artifacts/api-server`). Разбито по приоритету.

## 1. Кнопки и разделы-заглушки («скоро», без реального действия)

| Экран | Файл | Проблема |
|---|---|---|
| Мой профиль | `src/pages/profile/my-profile.tsx` | Кнопка «Пополнение Stars» — просто показывает тост «скоро», реального пополнения нет |
| Мой профиль | `src/pages/profile/my-profile.tsx` | Кнопка «История транзакций» — тост «скоро», список транзакций не открывается |
| Список чатов | `src/pages/chat/chat-list.tsx` | Пункт меню «Удалить историю» в контекстном меню чата — тост «скоро будет добавлено», реально ничего не удаляет |
| Настройки | `src/pages/settings/settings.tsx` → `/settings/rate` | «Оценить приложение» — статичный экран со звёздами, оценка никуда не сохраняется |
| Настройки | `src/pages/settings/settings.tsx` → `/chat/favorites` | «Избранное» открывает заглушку вместо реального списка избранных сообщений |
| Язык | `src/pages/settings/language.tsx` | Выбор языка не сохраняется и не влияет на интерфейс (весь интерфейс всё равно на английском/русском вперемешку) |
| Оформление | `src/pages/settings/appearance.tsx` | «Размер шрифта» и «Фон чата» — переключатели без реальной логики, ничего не меняют в чате |

**Что делать:** либо убрать эти пункты из UI до реализации, либо довести до реального функционала (особенно «Избранное» и «Удалить историю» — это базовые ожидания пользователя от мессенджера).

## 2. Функции, работающие только локально (не синхронизируются между устройствами)

| Функция | Файл | Проблема |
|---|---|---|
| Персоны (доп. аккаунты) | `src/pages/settings/personas.tsx` | Список персон хранится в `localStorage`, при смене устройства/очистке кеша пропадает |
| Автозагрузка медиа, «Очистить кэш» | `src/pages/settings/data.tsx` | Чисто локальное, ожидаемо — но стоит явно указать пользователю, что это применяется только на этом устройстве |
| Tor / Proxy / DoH | `src/pages/settings/connection.tsx` | Настройка реально влияет на сетевой слой, но хранится только в `localStorage` — при новом входе с другого устройства настройки сети сбрасываются |

**Что делать:** синхронизировать персоны и, по желанию, сетевые настройки через `PATCH /users/me/settings`, как это уже сделано для приватности и уведомлений.

## 3. Оплата и звонки — определить реальные ограничения

| Функция | Статус | Комментарий |
|---|---|---|
| Покупка Stars (`POST /stars/purchase`) | Демо-режим | Звёзды начисляются бесплатно, без реальной оплаты. Нужно решить: интегрировать платёжный шлюз (см. навык `monetization`) или явно обозначить в UI, что это демо-валюта |
| Premium-подписка | Работает через Stars | Полноценно списывает и хранит подписку, но так как Stars не покупаются за деньги — подписка тоже фактически бесплатна |
| Аудио/видеозвонки | Реализованы (WebRTC + сигналинг через WebSocket) | Это P2P-звонки без интеграции с телефонной сетью — это ожидаемо для мессенджера такого типа, дополнительных доработок не требуется, кроме UI-полировки |

**Что делать:** решить, нужна ли реальная монетизация Stars (тогда — подключать платёжный провайдер), или оставить как внутреннюю демо-экономику и явно это подписать в интерфейсе.

## 4. Отправка файлов и медиа в чате

**Файл:** `src/pages/chat/chat-window.tsx`

Загрузка голосовых сообщений работает полностью. Отправка фото/файлов через вложение реализована частично — `inputRef` на файлы существует, но полный путь «выбрать фото → предпросмотр → отправка со сжатием» не всегда доведён до конца для всех типов вложений. Рекомендуется:
- Добавить предпросмотр изображения перед отправкой
- Добавить прогресс загрузки для больших файлов
- Проверить обработку ошибок при превышении размера файла

## 5. Мелкие несостыковки

- **Экран входа** (`src/pages/auth/login.tsx`) — весь текст на английском («Decrypt Vault», «Identifier», «Cryptographic Seed», «Generate new keys»), в то время как остальное приложение на русском. Нужно перевести (см. документ по дизайну ниже — предлагается пересобрать экран входа полностью).
- **Быстрые реакции** в чате (`EMOJI_QUICK` в `chat-window.tsx`) — механизм реакций на сообщение есть на бэкенде, но выбор реакции через долгое нажатие работает не во всех состояниях экрана — нужно перепроверить на реальном устройстве.
- **Каналы** переиспользуют компонент создания групп (`/new-channel` → `CreateGroup`) без специфичных для канала полей (например, у канала не должно быть текстового описания «участники видят друг друга» и т.п. — нужно развести формы).

## 6. Рекомендованный порядок работ

1. Убрать или доделать заглушки из раздела 1 (быстро, но заметно пользователю).
2. Довести до конца отправку фото/файлов в чате (раздел 4) — это базовая функция мессенджера.
3. Определиться с монетизацией Stars/Premium (раздел 3) — бизнес-решение, а не только техническое.
4. Синхронизировать персоны между устройствами (раздел 2).
5. Редизайн экрана входа и обновление визуального стиля — см. `ДИЗАЙН_СИСТЕМА.md` и `ДИЗАЙН_ЭКРАНЫ.md`.
